## 概述

本文档详细描述了各个测试场景的具体输入和预期输出，用于手动测试或自动化测试验证。

### 通用请求头
所有测试都应包含以下基本请求头：
```
Content-Type: application/json
Authorization: Bearer test-token
```

### 通用响应头
成功响应应包含：
```
Sentoris-Trace-Id: <trace-id>
Sentoris-Signature: <signature>
Sentoris-Version: 1.0.0
```

## 基础请求

### 1. 非流式基础请求 - E2E-001

#### 输入
**HTTP方法**: POST  
**Endpoint**: `/v1/chat/completions`

**Headers**:
```
X-Sentoris-Session-ID: e2e-test-session-001
```

**Body**:
```json
{
  "model": "gpt-4o",
  "messages": [
    {"role": "user", "content": "Hello, how are you?"}
  ],
  "max_tokens": 100,
  "temperature": 0.7
}
```

#### 预期输出
**HTTP状态码**: 200 OK

**Response Headers**:
```
Content-Type: application/json
Sentoris-Trace-Id: <auto-generated>
Sentoris-Signature: <auto-generated>
```

**Response Body**:
```json
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "created": 1715000000,
  "model": "gpt-4o",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "I'm doing well, thank you for asking!"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 12,
    "completion_tokens": 15,
    "total_tokens": 27
  }
}
```

#### 数据验证检查项
| 检查项 | 位置 | 预期值 |
|--------|------|--------|
| HTTP状态码 | 响应头 | 200 |
| Trace ID | Sentoris-Trace-Id | 非空字符串 |
| 模型字段 | Response.model | "gpt-4o" |
| choices | Response.choices | 长度≥1 |
| usage | Response.usage | 存在 |
| Trace记录 | PostgreSQL.traces | 找到对应记录 |
| ExecutionState | PostgreSQL | "FINALIZED" |
| SessionID | PostgreSQL | "e2e-test-session-001" |

## 预算治理

### 1. 预算硬停止 - E2E-100

#### 前置条件
**Redis操作**: 设置会话预算
```
SET budget:e2e-test-session-100 0.005
```

#### 输入
**HTTP方法**: POST  
**Endpoint**: `/v1/chat/completions`

**Headers**:
```
X-Sentoris-Session-ID: e2e-test-session-100
Sentoris-Budget-Limit: 0.005
Sentoris-Budget-Strategy: hard_stop
```

**Body**:
```json
{
  "model": "gpt-4o",
  "messages": [
    {"role": "user", "content": "Generate a very long text..."}
  ],
  "max_tokens": 2000
}
```

#### 预期输出（预算耗尽场景）
**HTTP状态码**: 400 Bad Request 或 429 Too Many Requests

**Response Body**:
```json
{
  "error": {
    "code": "budget_exhausted",
    "message": "Budget limit exceeded: 0.005 USD",
    "param": null,
    "type": "invalid_request_error"
  },
  "sentoris": {
    "trace_id": "<trace-id>",
    "version": "1.0.0",
    "budget_remaining": 0.000
  }
}
```

#### Redis数据验证
| Redis Key | 预期值 |
|-----------|--------|
| `budget:e2e-test-session-100` | 0.005 |
| `budget:used:e2e-test-session-100` | ≤ 0.005 |
| `budget:e2e-test-session-100` ≥ `budget:used:e2e-test-session-100` | true |

### 2. 预算降级策略 - E2E-101

#### 输入
**Headers**:
```
X-Sentoris-Session-ID: e2e-test-session-101
Sentoris-Budget-Limit: 0.001
Sentoris-Budget-Strategy: degrade_model
```

**Body**:
```json
{
  "model": "gpt-4o",
  "messages": [{"role": "user", "content": "Hello"}],
  "max_tokens": 50
}
```

#### 预期输出
**HTTP状态码**: 200 OK

**Response Headers**:
```
Sentoris-Warning: Model degraded from gpt-4o to qwen-turbo due to budget constraints
```

**Response Body中model字段**: 应该是降级后的便宜模型（如"qwen-turbo"）

## 隐私脱敏

### 1. Masked级别隐私 - E2E-201

#### 输入
**Headers**:
```
X-Sentoris-Session-ID: e2e-test-session-201
Sentoris-Privacy-Level: masked
Sentoris-Privacy-Masked-Fields: $.messages[0].content
```

**Body**:
```json
{
  "model": "gpt-4o",
  "messages": [
    {"role": "user", "content": "My email is alice@example.com, phone: +1-555-123-4567"}
  ],
  "max_tokens": 50
}
```

#### PostgreSQL存储验证
**查询SQL**:
```sql
SELECT input FROM traces WHERE trace_id = '<trace-id>';
```

**预期input内容**（脱敏后）:
```json
{
  "prompt": "My email is ***@example.com, phone: +1-***-***-****",
  "metadata": {
    "pii_detected": true,
    "pii_types": ["email", "phone"],
    "privacy_level": "masked"
  }
}
```

或完全脱敏:
```json
{
  "prompt": "My email is ***, phone: ***",
  ...
}
```

#### 不应该出现在数据库中的内容
- ❌ `alice@example.com`
- ❌ `+1-555-123-4567`

---

### 2. Hash Only级别 - E2E-202

#### 输入
**Headers**:
```
Sentoris-Privacy-Level: hash_only
```

**Body**:
```json
{
  "model": "gpt-4o",
  "messages": [{"role": "user", "content": "Secret message"}],
  "max_tokens": 50
}
```

#### PostgreSQL存储验证
**预期input内容**:
```json
{
  "prompt_hash": "sha256:abcdef123456...",
  "metadata": {
    "privacy_level": "hash_only"
  }
}
```

## 可复现性

### 1. Strict可复现性 - E2E-302

#### 输入（请求1）
**Headers**:
```
X-Sentoris-Session-ID: e2e-test-session-302
Sentoris-Reproducibility: strict
```

**Body**:
```json
{
  "model": "gpt-4o",
  "messages": [{"role": "user", "content": "Random number: "}],
  "max_tokens": 20,
  "seed": 42,
  "temperature": 0
}
```

#### 输入（请求2，完全相同）
与请求1完全相同的Headers和Body

#### 预期输出
- 两次响应的 `choices[0].message.content` **必须完全相同**
- 两次响应的 `finish_reason` 必须相同

## Replay + Diff

### 1. 基础Replay - E2E-600

#### 前置条件
已有基线Trace，假设Trace ID为: `baseline-trace-001`

#### 输入
**HTTP方法**: POST  
**Endpoint**: `/v1/replay`

**Body**:
```json
{
  "baseline_trace_id": "baseline-trace-001",
  "model": "gpt-4o",
  "focus_fields": ["$.choices[0].message.content", "$.usage.total_tokens"]
}
```

#### 预期输出
**HTTP状态码**: 200 OK

**Response Body**:
```json
{
  "trace_id": "candidate-trace-001",
  "risk_report": {
    "baseline_trace_id": "baseline-trace-001",
    "candidate_trace_id": "candidate-trace-001",
    "model_changed": false,
    "focus_fields": ["$.choices[0].message.content", "$.usage.total_tokens"],
    "token_diff": {
      "similarity_ratio": 0.95,
      "edit_distance": 5,
      "baseline_length": 100,
      "candidate_length": 102
    },
    "risk": {
      "risk_level": "low",
      "recommendation": "approve",
      "methodology": "hybrid_edit_distance_model_change_v2",
      "risk_score": 0.05,
      "model_changed": false,
      "field_risks": [
        {
          "path": "$.choices[0].message.content",
          "old_value": "Hello world!",
          "new_value": "Hello world",
          "risk_level": "low",
          "confidence": 0.95,
          "change_type": "VALUE_CHANGE"
        }
      ],
      "reasons": ["Risk score is acceptable"]
    },
    "generated_at": "2024-05-02T12:00:00Z"
  }
}
```

#### PostgreSQL验证 - RiskReport表
**查询SQL**:
```sql
SELECT * FROM risk_reports 
WHERE baseline_trace_id = 'baseline-trace-001'
  AND candidate_trace_id = 'candidate-trace-001';
```

**预期记录存在**: YES

### 3. 高风险差异场景 - E2E-602

#### 前置条件
- 基线输出: "The answer is Yes"
- 候选输出: 完全不同的内容，如"Error occurred"或完全无关的回答

#### 预期RiskReport
```json
{
  "risk": {
    "risk_level": "high",
    "recommendation": "block_release",
    "risk_score": 0.85,
    "field_risks": [
      {
        "path": "$.choices[0].message.content",
        "risk_level": "high",
        "change_type": "VALUE_CHANGE"
      }
    ],
    "reasons": ["Risk score is critical (>= 0.7)"]
  }
}
```

#### UI显示验证
- ✅ 整体红色警告样式
- ✅ "High Risk"标签
- ✅ "BLOCK RELEASE"推荐
- ✅ 字段级差异高亮显示

## 错误处理

### 1. 无效模型 - E2E-500

#### 输入
**Body**:
```json
{
  "model": "invalid-model-name-xyz",
  "messages": [{"role": "user", "content": "Hello"}]
}
```

#### 预期输出
**HTTP状态码**: 400 Bad Request

**Response Body**:
```json
{
  "error": {
    "code": "invalid_model",
    "message": "The model 'invalid-model-name-xyz' does not exist",
    "type": "invalid_request_error"
  },
  "sentoris": {
    "version": "1.0.0"
  }
}
```

---

### 2. 版本不兼容 - E2E-501

#### 输入
**Headers**:
```
Sentoris-Version: 999.0.0
```

**Body**:
```json
{
  "model": "gpt-4o",
  "messages": [{"role": "user", "content": "Hello"}]
}
```

#### 预期输出
**HTTP状态码**: 400 Bad Request

**Response Body**:
```json
{
  "error": {
    "code": "version_mismatch",
    "message": "Unsupported Sentoris version: 999.0.0. Supported versions: 1.0.0",
    "type": "invalid_request_error"
  }
}
```

## 附录：数据验证快速参考

### Redis键名规范
| 键类型 | 键名格式 | 说明 |
|--------|----------|------|
| 总预算 | `budget:<session-id>` | Float, 单位USD |
| 预留预算 | `budget:reserved:<session-id>` | Float, 单位USD |
| 已用预算 | `budget:used:<session-id>` | Float, 单位USD |

### PostgreSQL表结构参考
#### traces表重要字段
```sql
SELECT 
  trace_id,
  session_id,
  parent_id,
  model,
  execution_state,
  input,
  output,
  constraints_applied,
  observations,
  proofs,
  created_at
FROM traces
WHERE trace_id = '<trace-id>';
```

#### risk_reports表重要字段
```sql
SELECT 
  baseline_trace_id,
  candidate_trace_id,
  model_changed,
  token_diff,
  risk_assessment,
  focus_fields,
  generated_at
FROM risk_reports;
```
